# Remote Work Syndicate — UI / Design System v1.0

Recommended file path:

`/docs/06-ui.md`

This document defines the visual direction, interaction principles, component system, and UX standards for Remote Work Syndicate.

Remote Work Syndicate should not look like a cheap job board, affiliate site, or “make money online” page.

It should feel like a modern, trustworthy, high-signal AI career intelligence company.

Reference feelings:

- Linear: clean, sharp, modern
- Perplexity: intelligent and calm
- Vercel: premium developer-tech feel
- Stripe: polished motion and trust
- Arc / Mercury: elegant product storytelling
- Levels.fyi: useful compensation intelligence
- G2 / NerdWallet: comparison and decision support

The final product should communicate:

> “This is the serious place to understand AI remote work platforms before applying.”

---

## 1. UI Philosophy

The interface should be:

1. **Trustworthy**
   - Clear disclosures
   - Verified dates
   - No fake urgency
   - No exaggerated income promises

2. **Modern**
   - Clean layout
   - Strong typography
   - Elegant cards
   - Smooth micro-interactions
   - Rich gradients used carefully

3. **Intelligence-led**
   - Show comparisons, signals, scores, insights, and recommendations.
   - Make users feel smarter after using the site.

4. **Content-first**
   - Pages must be highly readable.
   - Guides, platform pages, and comparisons should feel editorial and premium.

5. **Product-amplified**
   - The Platform Match Quiz should feel like a product experience, not a generic form.

6. **Visual storytelling**
   - Use sections that guide users from problem → insight → action.
   - Show the workflow visually: Discover → Compare → Prepare → Apply → Track.

7. **Conversion-aware**
   - Every major page should guide users to:
     - take the quiz
     - apply
     - subscribe
     - share outcome data

## 7. Motion and Animation

Motion should feel premium, not gimmicky.

### Animation principles

1. Motion should explain hierarchy.
2. Motion should create polish.
3. Motion should not slow the user down.
4. Motion should be subtle and fast.
5. Respect reduced-motion preferences.

### Recommended library

Use:

```text
framer-motion
```

Only if needed. CSS transitions are enough for many components.

### Motion patterns

#### Hero entrance

- Headline fades up.
- CTA fades up with slight delay.
- Floating platform cards animate subtly.
- Background gradient moves very slowly.

#### Card hover

- Slight lift: translateY(-4px)
- Border brightens.
- Soft shadow/glow increases.
- Transition: 180–220ms

#### Quiz transitions

- Step slides/fades.
- Progress bar animates.
- Results count up from 0 to match score.
- Recommended platform cards stagger in.

#### Data visualization

- Match score ring animates on load.
- Signal bars fill.
- Timeline steps animate as user scrolls.

#### Page transitions

Avoid heavy route transitions in V1. Keep navigation fast.

### Avoid

- Bouncy animations
- Slow parallax
- Excessive spinning/glowing objects
- Constant moving elements that distract from reading
- Animations on every scroll element

2. **Promise**
   - Remote Work Syndicate helps you find the right platform.

   - Take the quiz, compare platforms, browse jobs, read guides.

   - Verified platforms, last updated jobs, community hiring insights.

   - Start with Platform Match Quiz or browse jobs.

- Floating cards showing platforms and match scores

```text
Last verified today.
```

```text
Referral available. Disclosure applies.
```

```text
Based on community-submitted reports.
```

```text
No account required.
```

```text

```

```text

```

```text

```

- Clear

- Practical

## 28. Implementation Notes for Codex

Codex should:

1. Use Tailwind and shadcn/ui.
2. Build reusable components.
3. Use dark premium theme by default.
4. Use subtle motion with framer-motion only where needed.
5. Avoid overengineering animations.
6. Build accessible forms.
7. Keep pages SEO-friendly.
8. Use real product copy from this document.
9. Include referral disclosures.
10. Avoid scammy income-focused copy.
11. Make mobile experience excellent.
12. Use placeholder data that looks realistic but not fake.
13. Never invent fake community statistics.
14. Use “Coming soon” or “Early sample” where data is not available.

## 30. V1 UI Definition of Done

The UI is ready for V1 when:

1. The homepage clearly explains the product in under 10 seconds.
2. The platform pages feel trustworthy and useful.
3. The job pages clearly show verification and disclosure.
4. The Platform Match Quiz feels like a premium product experience.
5. The site looks excellent on mobile.
6. CTAs are clear and not overwhelming.
7. No page feels like a generic job board.
8. Motion improves the experience without distracting from content.
9. Referral disclosures are visible.
10. The design supports SEO content at scale.
